# Deep Dive Template

```
[AI Security Deep Dive] YYYY-MM-DD — TITLE

Source: LINK (author, date)

TL;DR — one sentence that makes someone want to read the rest.

What it is:
2–3 sentences. Plain language. What did the researchers find or build?

How it works:
Explain the attack or technique like you're drawing on a whiteboard.
Skip the math, keep the intuition.

So what?
Why should anyone outside academia care? Who's affected?

What to do about it:
Proposed defenses, mitigations, or "we don't have a good answer yet."

Bottom line:
One sharp sentence. Make it quotable.
```

## Tone Guide

- Write like a smart colleague breaking down a paper, not like you're summarizing an abstract.
- Use analogies to make technical concepts click. ("Think of it as picking a lock, except the lock is a neural network.")
- "So what?" is the most important section — if you can't explain why it matters, reconsider covering it.
- Always include defense/mitigation — never present attacks without context.
